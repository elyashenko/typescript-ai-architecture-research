# Contributing

Спасибо за интерес к этому исследованию! Вклад приветствуется.

## Как внести вклад

### 1. Добавить новый пример проекта

Если у вас есть интересный open-source проект с хорошей архитектурой:

1. Создайте issue с описанием проекта
2. Добавьте ссылку на GitHub
3. Опишите ключевые архитектурные решения

### 2. Улучшить документацию

- Исправление опечаток
- Улучшение примеров
- Добавление деталей
- Перевод на другие языки

### 3. Добавить код-примеры

- Новые patterns
- Real-world use cases
- Tool implementations

### 4. Создать диаграммы

- Архитектурные схемы
- Flow diagrams
- Сравнительные таблицы

## Structure Guidelines

### Добавление документации

Документация находится в `docs/`:
- Используйте четкую структуру заголовков
- Добавляйте примеры кода
- Включайте ссылки на источники

### Добавление примеров

Примеры находятся в `examples/`:
- Следуйте существующему стилю
- Добавляйте README для каждого примера
- Включайте package.json

### Создание диаграмм

Диаграммы находятся в `diagrams/`:
- Используйте Mermaid для визуализации
- Добавляйте пояснения
- Соблюдайте цветовую схему

## Code Style

- TypeScript с strict mode
- ESLint + Prettier
- Meaningful variable names
- Comments для сложной логики

## Pull Request Process

1. Fork repository
2. Create feature branch
3. Make changes
4. Add/update tests if applicable
5. Update documentation
6. Submit PR with clear description

## Questions?

Открывайте issue с вопросами или предложениями!
